# 01-工程初始化与脚手架.SPEC.md

## 目标
创建前后端项目骨架，约定目录结构、配置方式、公共类型与基础运行脚本。

## 环境约束（新增）
1. 后端 Python 环境必须使用 Miniconda（Conda）创建。
2. Python 版本固定为 3.10.x。
3. 本阶段需要提供可复用的 conda 环境定义文件（例如 `backend/environment.yml`）。

## 本阶段不做
1. 不做真实模型接入
2. 不做真实推理
3. 不做复杂业务逻辑

## 必做内容
### 后端
1. 创建 FastAPI 工程骨架
2. 创建配置模块
3. 创建数据库初始化模块（SQLite）
4. 创建 API 路由占位
5. 创建 services 目录与模型接口占位：
   - classifier_provider.py
   - segmentation_provider.py
   - measurement_provider.py
   - mock_providers.py
6. 提供 conda 环境文件并能一键创建 Python 3.10 环境

### 前端
1. 创建 Vue 3 + TypeScript + Vite 工程
2. 创建基础布局
3. 创建路由：
   - 患者信息页
   - 实时测量页
   - 历史记录页
4. 创建 API 调用与 WebSocket 占位模块

## 目录要求
后端必须预留 providers 目录或等价目录，用于隔离 Mock 与真实模型。

## 验收标准
1. 前后端工程可分别启动
2. 后端存在统一 Provider 接口文件
3. MockProvider 文件已创建
4. 前端基础页面可访问
