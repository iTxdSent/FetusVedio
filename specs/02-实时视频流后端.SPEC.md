# 02-实时视频流后端.SPEC.md

## 目标
跑通视频输入、帧读取、WebSocket 推流和基础状态管理。模型部分只使用 Mock。

## 环境约束（新增）
1. 运行本阶段后端服务时，必须在 Miniconda 的 Python 3.10 环境中执行。
2. 新增后端依赖时，必须同步更新 conda 环境定义文件。

## 本阶段模型要求
### 只允许使用 Mock
1. 不接真实分类模型
2. 不接真实分割模型
3. 不接真实测量算法

### 本阶段要实现的 Mock 行为
1. 每读取一帧，就调用 MockClassifierProvider
2. 当 plane != OTHER 时，调用 MockSegmentationProvider
3. 再调用 MockMeasurementProvider
4. 把模拟结果封装到统一 WebSocket 消息中

## 必做内容
1. 视频源抽象：
   - 采集卡源
   - 本地视频源
2. 最新帧缓存机制
3. WebSocket `/ws/stream`
4. 帧 JPEG 编码与推送
5. 当前会话状态机
6. Freeze / Unfreeze
7. 切换本地视频 / 恢复视频流
8. Mock 结果注入到消息包中

## Mock 输出建议
### 分类
可按帧号轮换输出：
- 0~59 帧：OTHER
- 60~119 帧：BV
- 120~179 帧：AAV
- 180~239 帧：3VT
- 240 帧后循环

### 分割
对目标切面返回固定几何图形：
- BV：两条圆/椭圆区域
- AAV：一条弯曲区域或简化多边形
- 3VT：两个圆形区域

### 测量
返回固定或轻微抖动的模拟数值：
- BV: SVC=3.1, IVC=4.0
- AAV: AAo=3.1, TA=2.9, Isthmus=2.3, DAo=3.2
- 3VT: DA=2.8, DAo=3.0

## WebSocket 结果结构
必须携带：
1. frame_index
2. image_base64
3. plane
4. confidence
5. metrics
6. qc_pass
7. source_state

## 验收标准
1. 没有任何真实模型文件时，WebSocket 仍能持续推送完整结果
2. 前端或调试客户端能看到模拟分类与测量结果
3. 切换视频源后仍能正常输出 Mock 结果
