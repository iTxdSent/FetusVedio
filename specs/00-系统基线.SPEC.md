# 00-系统基线.SPEC.md

## 目标
定义项目的长期稳定边界，避免后续开发跑偏。

## 环境基线（新增）
1. Python 环境隔离与包管理统一使用 Miniconda（Conda）。
2. Python 版本固定为 3.10.x。
3. 开发环境与 Ubuntu 部署环境都必须使用同一套 conda 环境定义（例如 `environment.yml`）。
4. 禁止把 `venv/virtualenv/poetry/pipenv` 作为项目标准环境方案。

## 系统定位
这是一个用于演示的胎儿超声实时解译 Demo，不是医院正式系统。

## 固定业务范围
1. 患者信息录入
2. 开始检查 / 结束检查
3. 采集卡视频显示
4. 本地视频替代输入
5. 切面分类结果显示
6. 分割与测量结果叠加显示
7. 自动留图
8. 手动保存
9. 历史查询

## 模型策略（本文件重点）
### 当前阶段模型策略
在真实模型接入前，系统必须基于 Mock 跑通全链路。

### Mock 范围
1. 切面分类模型：只做模拟输出
2. 分割模型：只做模拟输出
3. 测量后处理：只做模拟输出
4. 不做训练、不做导出 ONNX、不做真实推理性能优化

### Mock 的目标
1. 帮助前后端联调
2. 帮助验证接口契约
3. 帮助验证自动留图与历史查询链路
4. 为后续真实模型替换预留稳定边界

## 必须保留的替换点
1. 分类适配器接口
2. 分割适配器接口
3. 测量适配器接口
4. 模型配置文件位置
5. 统一结果结构

## 统一结果契约
### 分类输出
```json
{
  "plane": "AAV",
  "class_id": 2,
  "confidence": 0.93
}
```

### 分割输出
```json
{
  "success": true,
  "plane": "AAV",
  "mask": "<mock-mask-or-null>",
  "labels": [0, 1]
}
```

### 测量输出
```json
{
  "success": true,
  "plane": "AAV",
  "metrics": {
    "AAo": 3.1,
    "TA": 2.9,
    "Isthmus": 2.3,
    "DAo": 3.2
  },
  "qc": {
    "mask_nonempty": true,
    "topology_ok": true,
    "anatomy_rule_ok": true
  }
}
```

## 验收原则
在没有真实模型文件的情况下，系统也必须能够：
1. 启动
2. 显示视频
3. 输出模拟分类结果
4. 输出模拟分割叠加
5. 输出模拟测量值
6. 触发自动留图
7. 保存历史记录
