# 04-模型接入与测量叠加.SPEC.md

## 目标
完成“模型接口层 + Mock 实现 + 叠加绘制”的稳定架构。注意：本阶段仍然**不接入真实模型**。

## 环境约束（新增）
1. 模型接口层与推理编排服务必须运行在 Miniconda 的 Python 3.10 环境中。
2. 任何新增 Python 依赖必须更新 conda 环境定义，确保 Ubuntu 可复现部署。

## 本阶段核心任务
1. 定义统一接口：
   - ClassifierProvider
   - SegmentationProvider
   - MeasurementProvider
2. 提供默认 Mock 实现
3. 在推理编排器中通过依赖注入或配置选择 Provider
4. 完成叠加绘制：
   - 切面名称
   - 置信度
   - 分割轮廓/Mask 可视化
   - 测量值文字

## 严格禁止
1. 直接把 ONNX Runtime 写死在主流程里
2. 在控制器里直接写模型推理代码
3. 让前端依赖真实模型输出才能运行

## 推荐目录
```text
backend/app/services/providers/
├─ base.py
├─ mock_classifier.py
├─ mock_segmentation.py
├─ mock_measurement.py
└─ registry.py
```

## Provider 契约
### 分类
输入：frame_bgr  
输出：
```json
{
  "plane": "BV",
  "class_id": 1,
  "confidence": 0.91
}
```

### 分割
输入：frame_bgr, plane  
输出：
```json
{
  "success": true,
  "plane": "BV",
  "mask": "<numpy-array-or-equivalent>",
  "labels": [0, 1, 2]
}
```

### 测量
输入：frame_bgr, plane, mask  
输出：
```json
{
  "success": true,
  "plane": "BV",
  "metrics": {
    "SVC": 3.1,
    "IVC": 4.0
  },
  "qc": {
    "mask_nonempty": true,
    "topology_ok": true,
    "anatomy_rule_ok": true
  }
}
```

## 可替换性要求
未来替换为真实模型时，只允许修改：
1. 新增 RealClassifierProvider / RealSegmentationProvider / RealMeasurementProvider
2. 更新 registry 或配置文件
3. 尽量不改控制器、前端页面、数据库结构

## 验收标准
1. 使用 MockProvider 时，叠加层完整工作
2. 替换点清晰
3. 文档中明确标出“未来真实模型替换入口”
