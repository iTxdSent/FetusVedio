# 06-部署、联调与Demo验收.SPEC.md

## 目标
在 Ubuntu Demo 环境完成部署说明、运行脚本、联调和验收。

## 环境约束（新增）
1. Ubuntu 部署必须基于 Miniconda，并使用 Python 3.10 的 conda 环境。
2. 部署文档必须包含 conda 环境创建、激活、依赖安装与验证步骤。

## 本阶段模型说明
部署包默认只要求 Mock 能运行。真实模型接入留到后续替换阶段。

## 必做内容
1. Ubuntu 部署说明
2. 前后端启动脚本
3. 配置文件说明
4. Mock 运行模式说明
5. 真实模型替换说明文档：
   - 替换哪个目录
   - 替换哪个 Provider
   - 不需要改哪些模块

## 验收标准
1. 没有真实模型文件时仍能完整演示
2. 文档清楚说明如何从 Mock 迁移到真实模型
